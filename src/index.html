Static Site Boilerplate

Introduction

First and foremost, the purpose of building this Static Site Boilerplate, was to learn how to use Webpack. While learning Webpack, I figured I should build something fairly simple, and that I could potentially end up using in the future. Thus, the Static Site Boilerplate was born.

At the time of creating this; Webpack 4 was just barely released, and I had no knowledge of using any form of task runner ( Gulp & Grunt ), build tool/system, or module bundler. There was not a huge amount of documentation for Webpack 4 either. The use of the webpack.js.org next branch, the few Webpack 4 tutorials available at the time, previous version documentation and Webpack 2 & 3 tutorials were all I had to go on while creating this boilerplate.

I believe one of the more popular uses of Webpack is for dynamic web pages, mainly React. I wanted to create something for static websites that may have several pages rather than a single dynamic page or app.
